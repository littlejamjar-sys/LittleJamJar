# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in LittleJamJar, please report it responsibly by emailing security@littlejamjar-sys.dev instead of using the public issue tracker.

### Security Report Guidelines

Include the following information:
- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fixes (if any)

We will:
- Acknowledge receipt within 48 hours
- Investigate and assess the severity
- Develop and release fixes as quickly as possible
- Credit you in the security advisory (unless you prefer anonymity)

## Security Measures

This project implements the following security practices:

- **Dependency Scanning**: Automated vulnerability checks on all dependencies
- **Code Coverage**: Continuous code coverage monitoring to identify untested code paths
- **SBOM Generation**: Software Bill of Materials for supply chain transparency
- **Dependency Updates**: Regular automated dependency checks and updates

## Supported Versions

| Version | Status          |
|---------|-----------------|
| 1.x     | Supported       |
| < 1.0   | Not Supported   |

## Security Updates

Security updates will be released as patches to the current stable version. Users are encouraged to update immediately upon release.

## Contact

For security concerns, contact: security@littlejamjar-sys.dev
